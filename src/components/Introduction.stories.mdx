# DS components

DS components <u>agiliza el proceso de creación de producto, disminuyendo los costes asociados al desarrollo y mantenimiento de estos</u>. Mediante la creación de componentes de UI, se aísla la lógica comercial, los datos y el contexto de la aplicación, <u>facilitando el desarrollo y reutilización de los mismos</u>.
Los componentes, mejor conocidos como **historias**, participaran durante el desarrollo, las pruebas y el control de calidad.

**Storybook** es la herramienta utilizada para construir este sistema. Esta permite desarrollar con un proceso [**Component Driven**](https://componentdriven.org) que da como resultado componentes atómicos.

Es posible acceder a todas las historias mediante la navegación en la barra lateral. Para obtener información detallada sobre su funcionamiento, le recomendamos consultar la documentación correspondiente.
